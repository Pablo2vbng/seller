Módulo: Proyecto de Desarrollo de Aplicaciones Web


1. Introducción y Propósito

Seller es una plataforma web desarrollada para optimizar la gestión comercial de agentes de ventas. Su propósito principal es centralizar en una sola interfaz la gestión de clientes, la consulta de inventarios dinámicos y la elaboración de propuestas comerciales rápidas.

El diseño se basa en una estética de alto contraste ("Dark Mode") que minimiza la fatiga visual y proyecta una imagen tecnológica y profesional.

2. Metodologías y Estándares

Metodología BEM

Se ha implementado el estándar BEM (Block, Element, Modifier) para la arquitectura de clases CSS. Esta elección garantiza que cada componente visual sea independiente, facilitando la escalabilidad del proyecto y permitiendo que otros desarrolladores comprendan la jerarquía del código a simple vista.

Sass Avanzado

Para la gestión de estilos se utiliza Sass. Esto permite modularizar los tokens de diseño (colores, fuentes y espaciados) en archivos específicos, asegurando un sistema de diseño coherente y sin errores de ámbito global.

3. Stack Tecnológico

TypeScript & JS Vanilla

Uso de TypeScript para definir interfaces de datos y asegurar un tipado estático que reduce errores lógicos durante el desarrollo.

Flexbox, REM & Bootstrap

Layout construido íntegramente con Flexbox y unidades de medida relativas (REM), asegurando un escalado perfecto en cualquier resolución.Se ha integrado Bootstrap de forma selectiva, aprovechando sus capacidades nativas para componentes de interacción (como el menú Offcanvas)


4. Arquitectura de Datos Híbrida

La eficiencia de la aplicación reside en su modelo de datos dual, diseñado para maximizar el rendimiento:

  • Persistencia Crítica (MySQL): A través de un entorno XAMPP local, gestionamos el registro de usuarios y el alta oficial de clientes comerciales, asegurando la integridad de los datos.
  • Carga Dinámica (JSON): El catálogo de productos y tarifas se almacena en estructuras JSON. Esto permite realizar búsquedas instantáneas y filtrar precios sin necesidad de peticiones costosas al servidor de base de datos.

5. Funcionalidades del Sistema

  • Dashboard Interactivo: Una landing page diseñada con tarjetas de acceso rápido para minimizar clics innecesarios.
  • Preparador de Presupuestos: Sistema interactivo que permite añadir productos y calcular totales para generar ofertas de venta.
  • Temas de Interfaz: Switch dinámico de Modo Claro / Modo Oscuro gestionado mediante variables CSS nativas en el :root.
  • Responsive Nativo: Interfaz adaptable a tablets y smartphones para permitir el trabajo en movilidad de los comerciales.
